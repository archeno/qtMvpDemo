# Qt MVP 演示

这是一个展示在 Qt6 中实现的模型-视图-演示器 (MVP) 架构模式的演示项目。该应用程序提供了一个简单的 GUI，具有多个页面（下载、WebSocket、数据可视化），这些页面使用 MVP 模式演示不同的功能。

## 特性

- **下载页面**：演示数据下载功能。
- **WebSocket 页面**：展示 WebSocket 通信。
- **数据可视化页面**：显示数据可视化。
- 页面管理与生命周期处理（自动销毁、后台、手动关闭）。
- 模块化架构，将视图、演示器和服务层分开。

## 项目结构

- `main.cpp`：应用程序入口点。
- `MainWindow.*`：主应用程序窗口，具有页面导航。
- `PageFactory.*`：用于创建和管理页面的工厂。
- `pages/`：每个页面的视图类。
- `presenter/`：处理业务逻辑的演示器类。
- `service/`：数据操作的服务类。
- `data/`：数据模型。
- `devices/`：设备接口和实现。
- `parser/`：协议解析工具。

## 要求

- Qt6（带有 Widgets 模块）
- CMake 3.20 或更高版本
- C++17 兼容编译器（例如 MinGW、MSVC）

## 构建

1. 克隆仓库：
   ```bash
   git clone https://github.com/archeno/qtMvpDemo.git
   cd qtMvpDemo
   ```

2. 创建构建目录：
   ```bash
   mkdir build
   cd build
   ```

3. 使用 CMake 配置：
   ```bash
   cmake -G "MinGW Makefiles" ..
   ```
   （根据您的 Qt 设置使用适当的生成器）

4. 构建：
   ```bash
   cmake --build .
   ```

## 运行

执行构建的可执行文件：
```bash
./QtMVPDemo.exe
```

应用程序将显示一个主窗口，左侧是可用页面的列表，右侧是选定页面的内容。

## 架构

此项目遵循 MVP（模型-视图-演示器）模式：

- **视图**：Qt 小部件处理 UI 渲染和用户交互。
- **演示器**：在视图和服务之间调解，处理业务逻辑。
- **服务**：管理数据操作、网络调用等。

`PageFactory` 管理页面创建和生命周期，允许动态页面加载和内存管理。
